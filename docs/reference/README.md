# Reference Documentation

AI-readable and developer-facing reference guides for the koad:io ecosystem.

## Contents

| File | Description |
|------|-------------|
| `framework-layer.md` | Canonical reference for `~/.koad-io/` — what the framework provides, full directory tree with purposes, the `koad-io` CLI dispatcher, global commands, hook waterfall, what entities can use vs. must not modify |
| `packages-structure.md` | Complete guide to `~/.koad-io/packages/` — package inventory, dependency tree, file format, and navigation guide for agents arriving cold |
| `entity-directory-structure.md` | Canonical reference for the standard entity directory layout — every directory, every always-present file, the entity lifecycle, and the two-layer architecture as it manifests on disk |
| `entity-directory-structure-claude-section.md` | Shorter version of the above, formatted as a drop-in `## Directory Reference` section for new entity `CLAUDE.md` files |

## About This Section

Reference docs explain structure, not procedure. They answer "what is this and where do I find things" rather than "how do I do X." See `../guides/` for procedural guides and `../tutorials/` for step-by-step walkthroughs.
